# ADR 001: Добавление поддержки Streamable HTTP транспорта

## Статус

**Accepted** - Реализовано в v0.5.0

## Контекст

Текущая архитектура Pinchtab MCP Wrapper использует только stdio транспорт, что ограничивает возможности развертывания:

- Только локальное использование (AI агент должен быть на той же машине)
- Невозможность удаленного доступа к MCP серверу
- Сложности с облачными развертываниями и масштабированием
- Отсутствие поддержки serverless архитектур

MCP Protocol ввел новый транспорт **Streamable HTTP** (Revision 2025-03-26), который заменяет устаревший HTTP+SSE подход.

## Решение

Добавить поддержку Streamable HTTP транспорта с сохранением полной обратной совместимости со stdio.

### Выбранный подход

**Multi-transport архитектура** с фабрикой транспортов:

```
┌─────────────────────────────────────────────────────────────┐
│                 PinchtabMcpServer                           │
│                                                             │
│  ┌─────────────┐    ┌──────────────────┐    ┌────────────┐ │
│  │   Stdio     │    │ Streamable HTTP  │    │    SSE     │ │
│  │  (default)  │    │  (recommended)   │    │ (legacy)   │ │
│  └─────────────┘    └──────────────────┘    └────────────┘ │
│          │                    │                    │        │
│          └────────────────────┼────────────────────┘        │
│                               │                             │
│                    Transport Factory                        │
└─────────────────────────────────────────────────────────────┘
```

### Конфигурация

```bash
# Выбор транспорта
MCP_TRANSPORT=stdio              # По умолчанию
# MCP_TRANSPORT=streamable-http  # Для облачных развертываний

# Настройки HTTP
MCP_HTTP_PORT=3000
MCP_HTTP_HOST=0.0.0.0
MCP_AUTH_TYPE=bearer
MCP_AUTH_TOKEN=secret-token
```

### Архитектурные компоненты

1. **Transport Layer** - абстракция над различными транспортами
2. **Auth Layer** - Bearer/API-key авторизация для HTTP
3. **Session Management** - stateful/stateless режимы
4. **Security Layer** - CORS, rate limiting

## Альтернативы

### Альтернатива 1: Только stdio (текущее состояние)

**Плюсы:**
- Простота реализации
- Нет проблем с безопасностью

**Минусы:**
- Ограниченность локальным использованием
- Невозможность масштабирования

**Вердикт:** Отклонено - не соответствует требованиям облачных развертываний

### Альтернатива 2: SSE (Server-Sent Events)

**Плюсы:**
- Уже используется в MCP экосистеме
- Поддерживается SDK

**Минусы:**
- Deprecated в новой спецификации
- Менее эффективный, чем Streamable HTTP
- Двойное HTTP соединение (POST для команд, SSE для ответов)

**Вердикт:** Отклонено - устаревшая технология

### Альтернатива 3: gRPC

**Плюсы:**
- Высокая производительность
- Строгая типизация

**Минусы:**
- Не поддерживается MCP SDK
- Сложность настройки
- Проблемы с Web-интерфейсами

**Вердикт:** Отклонено - отсутствие поддержки в MCP

### Альтернатива 4: WebSocket

**Плюсы:**
- Real-time коммуникация
- Широкая поддержка

**Минусы:**
- Не входит в стандарт MCP
- Дополнительная сложность с переподключениями

**Вердикт:** Отложено - возможно в будущих версиях

## Последствия

### Положительные

1. **Облачные развертывания** - возможность деплоя на AWS/GCP/Azure
2. **Удаленный доступ** - AI агенты могут подключаться по сети
3. **Масштабируемость** - load balancing, multiple instances
4. **Serverless** - поддержка stateless режима
5. **Web UI** - возможность создания веб-интерфейсов

### Негативные

1. **Сложность безопасности** - необходимость авторизации и CORS
2. **Дополнительная конфигурация** - больше переменных окружения
3. **Увеличение кода** - ~500 строк нового кода
4. **Тестирование** - необходимость тестов для каждого транспорта

### Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Breaking changes в SDK | Средняя | Высокое | Пиним версию SDK, тестируем |
| Уязвимости безопасности | Средняя | Высокое | Обязательная авторизация |
| Снижение производительности | Низкая | Среднее | Бенчмарки, оптимизация |

### Фазы

1. **Подготовка** - обновление SDK, ADR

- **Security tests** - auth, CORS, rate limiting
